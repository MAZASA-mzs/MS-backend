# Система сбора наблюдений

Краткое описание сервиса: **Main Backend API** — это центральный микросервис системы сбора наблюдений. Он выполняет роль ядра бизнес-логики: управляет реляционной базой данных, обрабатывает единые профили пользователей, сохраняет наблюдения (фотографии и геометки), а также предоставляет интерфейс для администрирования команд. В рамках микросервисной архитектуры Backend работает в закрытом контуре и обслуживает исключительно gateway-сервисы (MAX Bot и TG Bot).

## Архитектура и зависимости

### Технологии и фреймворки:
- **Язык программирования:** Python 3.11+
- **Фреймворк:** FastAPI
- **База данных и ORM:** PostgreSQL, SQLAlchemy 2.0, Alembic
- **Кеш**: Redis
- **Контроль качества кода:** `flake8` (линтер), `black` (code style fixer), `pre-commit` (git hooks)
- **Тестирование:** `pytest`

### Взаимодействие с другими микросервисами:

Сервис полностью изолирован от публичного интернета. Он принимает HTTP-запросы только от доверенных gateway-микросервисов.
Авторизация межсервисного взаимодействия осуществляется путем передачи секретного токена в заголовке `X-Api-Key`. Состояние (State) и кэширование управляются на стороне gateway-ботов (через их собственные инстансы Redis), Backend отвечает только за персистентные данные.

### Используемые внешние сервисы:

- **Яндекс.Диск:** Используется в качестве S3-подобного хранилища для загрузки и хранения медиафайлов (фотографий наблюдений).

## Запуск сервиса

### Переменные окружения (.env)

Создайте файл `.env` в корне проекта (на основе `.env.example`):
```env
POSTGRES_USER=postgres
POSTGRES_PASSWORD=postgres
POSTGRES_DB=observations
POSTGRES_HOST=db
POSTGRES_PORT=5432
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_DB=0
API_KEY=app_secret_key
YANDEX_DISK_TOKEN=yandex_token
```

### Запуск

1. Установите зависимости: `pip install -r requirements.dev.txt`
2. Примените миграции: `alembic upgrade head`
3. Запустите необходимую инфраструктуру: `docker compose up redis postgres`
4. Запустите сервер: `uvicorn app.main:app --host 0.0.0.0 --port 8089 --reload`

## API документация

После успешного запуска приложения, интерактивная документация (OpenAPI / Swagger) доступна по адресу:
- **Swagger UI:** `http://localhost:8089/docs`

## Как тестировать

В проекте настроены Unit-тесты с использованием `pytest`. Запуск тестов интегрирован в git hooks (через `pre-commit`), что предотвращает коммиты с падающими тестами или нарушениями стайл-гайдов.

**Запуск тестов локально:**
```bash
pytest -v
```
**Ручной запуск линтеров и фиксеров:**
```bash
black .
flake8 .
```
